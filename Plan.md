# 비원어민 한국어 발음 STT 오류 기반 국적(모국어) 분류 프로젝트 계획서

본 프로젝트는 Whisper 등 STT(Speech-to-Text) 모델을 활용하여, 한국어가 모국어가 아닌 외국인이 발음하는 한국어 음성을 텍스트로 변환하고 **그 과정에서 발생하는 인식 오류(오발음/전사 오류)의 패턴을 분석하여 해당 화자의 모국어(국적)를 분류 및 분석**하는 것을 목표로 합니다.

---

## 1. 프로젝트 개요

*   **목표:** 비원어민 한국어 음성 데이터를 STT 모델에 통과시킨 뒤, 오류(STT결과 vs 실제 정답) 데이터를 활용하여 화자의 모국어를 분류/예측하는 파이프라인 구축.
*   **활용 데이터:** AI Hub의 '교육용 각종 언어 모국어 사용자의 한국어 음성 데이터' (영어 71469, 아시아어 71479, 유럽어 71489 등)
*   **핵심 가설:** **"모국어 발성 습관(Phonology)의 차이가 STT 모델의 인식 오류 패턴에 일관되게 나타날 것이다."** (예: 영어권은 'ㄹ'과 'ㄴ' 발달에 따른 특정 오탈자 빈도 증가, 아시아권은 성조나 특정 모음 오류 등)

---

## 2. 파이프라인 구현 계획 (Phase 1 ~ Phase 4)

### Phase 1: 데이터 전처리 (Data Preprocessing)
AI Hub의 방대한 원천 음성 데이터(.wav/.pcm)와 라벨링 데이터(.json)를 파이프라인에 맞게 정제합니다.

1.  **JSON 메타데이터 파싱:** 각 음성 파일의 정답 전사 텍스트(Ground Truth) 및 화자의 메타데이터(국적/모국어, 성별, 나이 등) 추출.
2.  **데이터 클렌징:** 잡음이 너무 심하거나 길이가 너무 짧은 오디오 필터링. 전체 경로 및 라벨을 포함한 DataFrame(또는 CSV/Arrow 형식) 구축.
3.  **샘플링:** 초기 실험을 위해 각 국가별(영어권, 아시아권, 유럽권) 균일한 수량을 샘플링. (예: 각 권역별 1,000~5,000 문장 추출)

### Phase 2: STT 모델 추론 (STT Inference)
준비된 음성을 STT 모델을 통해 텍스트로 변환합니다.

1.  **모델 선정:** 초기 Baseline으로 `OpenAI Whisper` (large 모델) 사용.
2.  **배치 추론(Batch Inference):** 샘플링된 음성을 STT 모델에 입력하여 모델의 추론(Prediction) 텍스트를 추출. 
3.  **결과 저장:** `[음성 파일명, 모국어(정답), 실제 발화 텍스트(GT), STT 추론 텍스트(Pred)]` 형태로 Mapped Data 저장.

### Phase 3: 오류 분석 및 특징 추출 (Error Analysis & Feature Extraction)
실제 텍스트(GT)와 STT 추론 텍스트(Pred) 간의 차이를 정량화하여 모델 입력용 특징(Features)을 생성합니다.

1.  **CER/WER 계산:** Character Error Rate(글자 단위 인식률) 등을 통해 국가별 STT 성능 차이 1차 분석.
2.  **음운/형태소 단위 정렬(Alignment):** 정답 글자와 STT 글자를 줄기세포로 비교하여 **어느 글자가 어떻게 잘못되었는지(오류 쌍)**를 통계화. (예: `["설명" -> "설명", "입니다" -> "입미다"]`)
    *   한국어 형태소 분석기(Kiwi 등)나 자모 분리 라이브러리(hgtk)를 사용해 초/중/종성 단위로 오답 패턴을 분석하면 훨씬 더 유의미한 피처(Feature)를 얻을 수 있습니다.
3.  **오류 특징 벡터화 (Feature Vectorization):** 오류 발생 유형(삽입, 삭제, 대치) 빈도 및 TF-IDF 기반의 오류 N-gram 벡터화 적용.

### Phase 4: 국적 분류기 학습 (Nationality Classification)
분석된 STT 텍스트와 오류 패턴을 바탕으로, 화자의 모국어를 예측하는 분류 모델을 학습시킵니다.

1.  **입력 피처 구성:**
    *   *Approach A:* 직접 추출한 오류 특징(Feature) 벡터를 Random Forest / XGBoost 과 같은 ML 모델에 입력.
    *   *Approach B:* STT 모델이 추론한 텍스트 자체(에러가 포함된 텍스트)를 한국어 사전학습 언어모델(KoBERT, RoBERTa 등)에 넣어서 직접 국적을 분류하도록 Fine-tuning.
2.  **모델 학습 및 평가:** 학습/검증 데이터 분리 후 정확도(Accuracy), F1-Score 확인.

---

## 3. 검토가 필요한 사항 (Open Questions)

본 프로젝트를 본격적으로 시작하기 앞서, 아래 항목들에 대한 방향성을 확인해주세요.

1.  **데이터 규모:** 개발 초기에는 전체 데이터의 일부(예: 10,000개 정도)만 샘플링하여 파이프라인을 구축하고 점진적으로 늘려가는 방식을 추천드립니다.
2.  **STT 모델 종류:** `Whisper` 오픈소스 모델을 첫 Baseline으로 삼고, 추후 네이버 Clova나 Google Cloud STT 등 상용 API 성능과 비교할지를 결정해 주시면 좋습니다.
3.  **접근 방식(Approach):** Phase 4에서 **A(규칙 기반의 단어/음운 빈도 추출 + ML 자동 분류)** 와 **B(문장 자체를 통째로 넣는 딥러닝 BERT 분류)**를 순차적으로 모두 진행하여 결과를 비교해 보는 것을 제안합니다.
